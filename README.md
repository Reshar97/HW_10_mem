Механизмы управления памятью в ядре Linux (HW_10_mem)

Этот репозиторий содержит реализацию различных механизмов выделения памяти в ядре Linux.

Описание

Проект включает в себя пять модулей ядра, каждый из которых демонстрирует работу определенного аллокатора памяти:

    ex_kmalloc.c — выделение непрерывного блока физической памяти.
    ex_vmalloc.c — выделение блока в виртуальном адресном пространстве.
    ex_kmem_cache.c — использование SLAB-аллокатора для кэширования объектов фиксированного размера.
    ex_mempool.c — использование пула памяти (mempool) для гарантированного выделения ресурсов.
    ex_get_page.c — низкоуровневое выделение страниц физической памяти.

Каждый модуль измеряет время, затраченное на операцию выделения, и выводит результат в системный лог (dmesg).

Требования

Для сборки модулей необходимо наличие установленных заголовков ядра:

	sudo apt install linux-headers-$(uname -r) build-essential

(или аналогичная команда для вашего дистрибутива)


Сборка и запуск

В репозитории предоставлен Makefile для автоматизации процесса.

Сборка модулей:

    	make

В результате будут созданы файлы .ko в текущей директории.
Загрузка модулей в ядро:

    	sudo make load

После выполнения команды проверьте вывод системного журнала:

    	dmesg | tail -n 50

Вы увидите результаты замеров времени для каждого аллокатора.
Выгрузка модулей:

    	sudo make unload

Очистка (удаление файлов сборки):

    	make clean

Проверка (сборка, загрузка, вывод лога, выгрузка):
    
    	make check
